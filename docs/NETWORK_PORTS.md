# 📊 Матрица сетевых портов платформы  

Ниже приведена строгая маршрутизация портов для локальной разработки.  
*Правило Actuator:* Служебные метрики сервиса всегда доступны на порту `Порт сервиса + 100`.  

### 🌐 Business API & Actuator
| Микросервис          | HTTP Порт | Actuator Порт | Назначение                            |
|:---------------------|:----------|:--------------|:--------------------------------------|
| **API Gateway**      | `8080`    | `8180`        | Единая точка входа (Routing)          |
| User Service         | `8081`    | `8181`        | Управление пользователями             |
| Delivery Service     | `8082`    | `8182`        | Оркестрация заказов и доставок        |
| Transport Service    | `8083`    | `8183`        | Управление автопарком и водителями    |
| Payment Service      | `8084`    | `8184`        | Расчет стоимости и платежи            |
| Cargo Service        | `8085`    | `8185`        | Управление товарами (Каталог/Резервы) |
| Notification Service | `8086`    | `8186`        | Уведомления (Email, SMS)              |
| Reporting Service    | `8087`    | `8187`        | Генерация аналитики                   |

### 🗄️ Базы данных & Инфраструктура  
| Сервис                | Порт    | Тип       | Примечание                                        |
|:----------------------|:--------|:----------|:--------------------------------------------------|
| **PostgreSQL (Main)** | `5433`  | Контейнер | Основная СУБД для бизнес-сервисов                 |
| **PostgreSQL (Auth)** | `5434`  | Контейнер | СУБД для Keycloak                                 |
| **MongoDB**           | `27018` | Контейнер | Хранилище документов и медиа (GridFS)             |
| **Redis**             | `6379`  | Контейнер | In-memory кэш                                     |
| **DbGate**            | `5051`  | Контейнер | Веб-интерфейс для управления PostgreSQL и MongoDB |

### 🔐 Identity, Admin & Observability  
| Сервис            | Порт   | Назначение                            |
|:------------------|:-------|:--------------------------------------|
| **Keycloak**      | `8080` | Identity & Access Management (SSO)    |
| **Kafka UI**      | `8090` | Веб-интерфейс для брокера сообщений   |
| **Prometheus**    | `9090` | Сбор и агрегация метрик               |
| **Kafka**         | `9092` | Брокер сообщений                      |
| **Grafana**       | `3000` | Дашборды мониторинга                  |
| **Loki**          | `3100` | Агрегация логов                       |

### 🗄️ Доступ к базам данных через DbGate  
| База данных    | Подключение через DbGate                                        |
|:---------------|:----------------------------------------------------------------|
| **PostgreSQL** | `postgres:5432` (внутри Docker) или `localhost:5433` (с хоста)  |
| **MongoDB**    | `mongodb:27017` (внутри Docker) или `localhost:27018` (с хоста) |

**DbGate доступен по адресу:** [`http://localhost:5051`](http://localhost:5051)  

#### После первого запуска добавь подключения вручную:  

| База данных    | Параметры подключения                                                                                             |
|:---------------|:------------------------------------------------------------------------------------------------------------------|
| **PostgreSQL** | Хост: `postgres`<br>Порт: `5432`<br>БД: `logistics_platform_db`<br>Пользователь: `postgres`<br>Пароль: `postgres` |
| **MongoDB**    | Хост: `mongodb`<br>Порт: `27017`<br>БД: `logistics_cargo_mongo_db`<br>Без аутентификации                          |

> **Важно:** Используй имена сервисов (`postgres`, `mongodb`) как хост, 
потому что DbGate работает внутри сети Docker.

**Запросы отправлять отсюда:** [`https://hoppscotch.io/`](https://hoppscotch.io/)
