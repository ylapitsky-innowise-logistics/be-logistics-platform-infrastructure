# 📘 Шпаргалка: Рабочий флоу <br>в GitHub Projects **`Logistics Platform Development`**
Этот мануал поможет вспомнить, как правильно вести задачи и автоматизировать движение карточек по доске (Todo ➡️ In Progress ➡️ Review ➡️ Done).

## 📋 Шаг 1. Планирование и запуск задачи

1. **Создание:** Зайди на Канбан-доску организации и внизу колонки **Todo** нажми `+ Add item`.

2. **Черновик:** Напиши краткую суть задачи и нажми `Enter` (создастся локальный Draft).

3. **Привязка к микросервису:** Нажми на созданный черновик ➡️ кликни **Convert to issue** ➡️ выбери нужный репозиторий (например, `user-service`, `cargo-service` или `infrastructure`). Карточка превратится в официальную задачу.



## 🌿 Шаг 2. Взятие в работу (In Progress)

_Не создавай ветку в IDE вручную. Делай это прямо из интерфейса GitHub:_

1. Кликни на карточку задачи на доске.

2. В правом боковом меню найди раздел **Development** и нажмите **Create a branch**.

3. Добавь префикс к названию ветки: `feat/` (для новых фич) или `fix/` (для багов). Нажми **Create branch**.

4. GitHub покажет всплывающее окно. **Скопируй команду** (вида `git fetch origin...`) и вставь её в терминал IntelliJ IDEA, чтобы автоматически скачать и переключиться на нужную ветку.

5. Перетащи карточку на доске в колонку **In Progress**.



## 🔗 Шаг 3. Написание кода и код-ревью (Review)

1. Напиши код в IDEA, сделай коммит и запушь ветку на GitHub.

2. Зайди в репозиторий на GitHub и нажми **Create pull request**.

3. **Магические слова (Важно!):** В описании Pull Request обязательно напиши фразу:

   > **`Closes #НОМЕР_ЗАДАЧИ`** (например: `Closes #15`).

4. Автоматика GitHub Projects сама считает эту фразу и мгновенно перекинет твою карточку в колонку **Review**.



## 🚀 Шаг 4. Слияние кода и завершение (Done)

1. Убедись, что тесты прошли удачно.

2. Нажми зеленую кнопку **Merge pull request** в ветку `main`.

3. **Финал:** За счет магического слова `Closes` пулл-реквест сливается, связанная задача автоматически закрывается в репозитории, а карточка на Канбан-доске **сама улетает в колонку Done**. Флоу завершен!

---
> 📌 **Памятка по префиксам веток:**
> - `feat/short-name` — новый функционал, эндпоинты, логика.
> - `fix/short-name` — исправление багов, конфигов, портов.
> - `infra/short-name` — настройка docker, kafka, баз данных.
