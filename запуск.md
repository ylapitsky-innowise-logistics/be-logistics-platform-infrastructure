## Запуск инфраструктуры

```bash


# Запустить все сервисы инфраструктуры
docker-compose up -d

# 2️⃣ При зависании сервиса
# Перезапустить (быстро)   -   Контейнеры уже запущены, нужно просто перезагрузить их
docker-compose restart mongodb
# или
docker-compose restart # 4️⃣ Нужно просто "перезагрузить всё"

#   Перезапустить конкретный сервис   -   Только один сервис (например, PostgreSQL)
docker-compose restart postgres

# 1️⃣ После изменения docker-compose.yml
# Остановить и запустить заново   -   Полный цикл: остановить + удалить контейнеры + запустить свежие
docker-compose down
docker-compose up -d

# 3️⃣ При смене переменных окружения (паролей и т.д.)
# Остановить + удалить данные + запустить   -   Полный сброс (очистка БД)
docker-compose down -v    # ⚠️ удалит данные!
docker-compose up -d

# Проверить статус
docker-compose ps

# Посмотреть логи конкретного сервиса
docker-compose logs -f postgres

# Остановить инфраструктуру
docker-compose down

# Остановить с очисткой данных
docker-compose down -v
```
